# RepoScale Visibility Launch Design

**Positioning:** Alpha experimental, benchmark-serious.

## Goal

Improve RepoScale's public presentation so the repository is easier to understand, easier to share, and more likely to convert visits into stars, contributors, and qualified early adopters.

## Audience

- AI eval builders looking for a runnable benchmark
- OSS contributors who can add cases or improve scoring
- Researchers and agent-tool builders evaluating repo understanding

## Message

### Primary line

RepoScale is an alpha benchmark for repo continuation intelligence.

### Supporting line

Evaluate how well models read existing software projects, infer intent, detect gaps, and propose coherent next steps.

### Proof points

- 12 curated cases
- 3 benchmark tracks
- Reproducible CLI pipeline
- Judge stability scoring

## Package

### README

Restructure the README into a faster narrative:

1. Hero
2. Why it matters
3. What you get today
4. Quickstart
5. Visual proof
6. Tracks and capabilities
7. Alpha caveats
8. Contribution path

### Visuals

Add two lightweight SVG assets:

- Social preview image for GitHub and link sharing
- README overview graphic showing the pipeline and alpha proof points

### Launch kit

Create a reusable launch document with:

- GitHub release title and body
- X post
- LinkedIn post
- Repo topics
- Public checklist for making the repository discoverable

## Constraints

- Keep claims honest: alpha, not benchmark-grade stable
- Avoid dark-pattern hype language
- Use assets that live in the repo and can be reused in docs and release notes
- Optimize for clarity over ornament
