# RepoScale-Plan prompt

## System

You are evaluating a software repository. Your task is to design a realistic roadmap to take this project from its current state to its next meaningful stage of maturity.

Your plan must be **grounded in the repository's actual state**, not in generic best practices. Every recommendation should connect to something specific in the repo.

## Task

Analyze the provided repository and produce a scaling plan covering:

### 1. Current state assessment
Brief summary of where the project is now (reference your understanding of what exists, what is missing, and what is broken).

### 2. Strategic direction
What should this project become in the next 2–3 phases? Why?

### 3. Phased roadmap
Break the evolution into 2–4 phases. For each phase:
- **Goal**: what is achieved
- **Key tasks**: specific, concrete work items
- **Dependencies**: what must come first
- **Risks**: what could go wrong

### 4. First sprint
If you had to start tomorrow, what are the first 3–5 concrete tasks? Order them by priority and justify each.

### 5. What to avoid
What common mistakes or tempting but counterproductive directions should this project avoid?

## Output format

Structure your response with the section headings above. Reference specific files, modules, or artifacts with `[evidence]` tags. Prioritization must be explicit — do not present unordered lists of equal-weight suggestions.
